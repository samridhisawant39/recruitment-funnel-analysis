# Recruitment Funnel Analysis & Hiring Insights

## Problem

Recruitment teams often lack visibility into where candidates drop off in the hiring pipeline, leading to inefficiencies and delays in hiring decisions.

## Objective

To analyse candidate pipeline performance and identify bottlenecks across hiring stages using SQL and Tableau.

## Tools Used

* SQL (MySQL Workbench)
* Tableau
* Excel

## Dataset

The dataset includes:

* Candidate ID
* Application date
* Interview date
* Offer date
* Source of application
* Experience level
* Candidate status

## Key Analysis

* Candidate distribution by status
* Candidate distribution by source
* Experience level breakdown
* Average time from application to interview
* Average time from interview to offer
* Monthly application trends
* Candidate status distribution by source

## Key Insights

* Majority of candidate drop-offs occur at the interview stage
* LinkedIn and referral channels generate the highest number of candidates
* The interview stage takes longer than the offer stage, indicating process delays
* Application volume shows a peak followed by a sharp decline in the final month

## Files in this Repository

* `analysis.sql` → SQL queries used for analysis
* `hiring_clean_final.csv` → cleaned dataset
* `Dashboard 1.png` → Tableau dashboard screenshot

## Summary

This project demonstrates the use of SQL for analysing recruitment data and Tableau for visualising hiring funnel performance, helping identify inefficiencies and improve decision-making in hiring processes.
